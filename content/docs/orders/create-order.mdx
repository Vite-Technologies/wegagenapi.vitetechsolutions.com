## Endpoint

`POST https://pranaconnect.wegagenbanksc.com.et/api/orders/create-order`

Create a new order request to initiate a real-time transaction route across Wegagen Bank's infrastructure. When an order is created, the middleware triggers a payment according to the selected payment method.

<Callout title="Webhook requirement" type="info">
  `notify_url` is required because this flow depends on asynchronous transaction
  updates being delivered back to your backend.
</Callout>

- Authentication requires `x-secret-key` on every request
- `notify_url` is required for asynchronous transaction updates
- Supported payment methods are `paybill` and `ussd_ebirr`

## Body Parameters

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `bill_code` | `string` | Yes | Unique bill code or merchant payment identifier used within mobile banking channels |
| `customerName` | `string` | Yes | Full name of the customer making the payment |
| `totalAmount` | `number` | Yes | Total order amount represented as a flat numeric value such as `1500` |
| `customerPhone` | `string` | No | Customer phone number. E.164 format is preferred, for example `+251911234567` |
| `cashierId` | `string` | Yes | Identifier for the cashier, terminal, or initiating retail point |
| `notify_url` | `string` | Yes | Webhook callback URL on your server for asynchronous status notifications |
| `order_id` | `string` | Yes | User-defined unique reference generated by your platform for this invoice |
| `payment_method` | `enum` | Yes | One of `paybill` or `ussd_ebirr` |
| `services` | `array<object>` | Yes | List of service or item records included in the order |
| `order_tags` | `json` | No | Flat metadata key-value pairs only. Keys cannot start with `$` or contain dots. Values must be flat strings or numbers |

## `services` Object

| Field | Type | Description |
| --- | --- | --- |
| `name` | `string` | Name of the item or service |
| `price` | `number` | Unit price of the item |
| `quantity` | `number` | Quantity of the item |
| `description` | `string` | Additional service details |

## Notes

- `notify_url` should point to a secure backend webhook endpoint that your platform controls
- `payment_method` currently supports `paybill` and `ussd_ebirr`
- Use `order_tags` for flat merchant metadata only
